## 1. プロジェクト概要

求人サービスのコーポレートHP。Vercel + Neon を基盤とした商用グレードの構成で、  
フォーム投稿受付機能を持ち、将来のスケールアップに耐えるアーキテクチャで構築する。

> **コンテンツ**: 初期リリースはモック文言で実装する。本番コンテンツへの差し替えは別途調整。

---

## 2. 機能要件

### 2.1 ページ構成（初期リリース）

| ページ | パス | 概要 |
|---|---|---|
| ランディングページ | `/` | ヒーローセクション、求人サービス紹介、CTA |
| About | `/about` | 会社・チーム紹介 |
| お問い合わせ | `/contact` | フォーム投稿 |
| サンクスページ | `/contact/success` | フォーム送信完了 |
| プライバシーポリシー | `/privacy-policy` | 個人情報の取り扱い（フォームの同意リンク先）|

### 2.2 フォーム投稿機能

#### 入力項目
| フィールド | 型 | 必須 | バリデーション |
|---|---|---|---|
| 氏名 | text | ○ | 1〜100文字 |
| メールアドレス | email | ○ | RFC 5322 準拠 |
| 件名 | text | ○ | 1〜200文字 |
| 本文 | textarea | ○ | 1〜2000文字 |
| プライバシーポリシー同意 | checkbox | ○ | true のみ |

#### 処理フロー
1. クライアントサイドバリデーション（Zod + React Hook Form）
2. Server Action 経由で DB に保存
3. 管理者メール通知（Resend）
4. 投稿者への自動返信メール
5. `/contact/success` へリダイレクト（サンクスページ）

### 2.3 3D インタラクション（WANT）

3D 演出はコンセプトごとに異なる。A/B テスト後に勝者コンセプトの仕様を残す。

| コンセプト | 3D 演出 | モバイル代替 |
|---|---|---|
| A: CONNECTED | Particle Network（ノードと線のつながり表現） | CSS グラデーション静止背景 |
| B: BOLD | Wireframe Icosahedron（ワイヤーフレーム正二十面体の回転） | CSS グロー効果の静的 SVG |
| C: HUMAN | Floating Spheres（暖色の半透明球体が浮遊） | CSS `@keyframes float` アニメーション |

> モバイル (< 768px) では全コンセプトとも WebGL 無効。スクロール連動演出は Desktop のみ。

## 3. 非機能要件

### 3.1 パフォーマンス

| 指標 | 目標値 | 計測方法 |
|---|---|---|
| LCP | ≤ 2.5s | Core Web Vitals |
| FID / INP | ≤ 200ms | Core Web Vitals |
| CLS | ≤ 0.1 | Core Web Vitals |
| Lighthouse Score | ≥ 90 (全項目) | Vercel Speed Insights |
| 3D Canvas FPS | ≥ 60fps (Desktop) | - |

> 3D コンポーネントは `next/dynamic` + `{ ssr: false }` でコード分割し、LCP に影響させない。

### 3.2 可用性・信頼性

| 項目 | 要件 |
|---|---|
| 稼働率 | 99.9%（Vercel SLA に準拠） |
| DB 接続 | Neon Serverless Driver によるコネクションプーリング |
| フォームデータ永続性 | DB 保存完了をもってレスポンス返却 |

### 3.3 セキュリティ（→ 詳細は security.md）

- OWASP Top 10 対応
- 入力値の完全なサーバーサイドバリデーション
- レート制限（フォーム投稿: 10回/時/IP）
- HTTPS 強制（Vercel デフォルト）

### 3.3b アクセシビリティ

- 準拠レベル: **WCAG 2.1 AA**
- キーボード操作: 全インタラクティブ要素が Tab / Enter / Escape で操作可能
- スクリーンリーダー: ランドマーク・`aria-label`・`alt` テキストを必須実装
- カラーコントラスト: テキスト 4.5:1 以上 / 大テキスト 3:1 以上（Lighthouse で計測）
- アニメーション: `prefers-reduced-motion` 検知時は全アニメーション無効化
- 3D Canvas: `aria-hidden="true"` を付与（装飾要素として扱う）

### 3.3c A/B テスト

| 項目 | 仕様 |
|---|---|
| バリアント数 | 3（A: CONNECTED / B: BOLD / C: HUMAN） |
| 割り当て方法 | Middleware で初回アクセス時に Cookie 付与（33% 均等分配） |
| 保持期間 | Cookie: 30日 |
| 計測指標 | バリアント別お問い合わせコンバージョン率 |
| 判定条件 | 最低4週間 / バリアントあたり30件で統計判定 |
| 詳細 | → `design.md §7` 参照 |

### 3.4 保守性

| 項目 | 方針 |
|---|---|
| TypeScript | strict モード必須 |
| コンポーネント | 単一責任・Props 型定義必須 |
| テスト | ユニット: Vitest、E2E: Playwright |
| Lint / Format | ESLint (Next.js 推奨) + Prettier |
| CI | GitHub Actions（lint → type-check → test → build） |

### 3.5 拡張性

- ルートグループ `(marketing)` / `(dashboard)` で関心分離
- 認証追加時は `(dashboard)` グループに middleware を適用するだけで対応可能
- 多言語対応（i18n）: next-intl を想定した URL 設計（`/ja/`, `/en/`）

## 7. 法的要件チェックリスト

> **前提**: 日本法（APPI・職業安定法・景品表示法・電気通信事業法・特定電子メール法）を適用。  
> リリース前に法務担当または弁護士と確認すること。

### 致命的（リリース前に法務確認・対応が必須）

| # | 項目 | 確認事項 | 参照 |
|---|---|---|---|
| 1 | **プライバシーポリシー コンテンツ確定** | APPI 必須記載事項（取扱事業者・利用目的・委託先・保持期間・請求窓口）を法務確認のうえ確定する。「同意チェックボックス」の有効性はリンク先PPの内容に依存する。 | `implementation-guide.md §9.5` |
| 2 | **職業安定法 許可要件の確認** | サービスが「求人情報掲載のみ」か「有料職業紹介（マッチング・紹介）」かを確定し、後者であれば厚生労働省の有料職業紹介事業許可証が必要。無許可での紹介行為は6ヶ月以下の懲役または30万円以下の罰金。 | 法務担当 |
| 3 | **電気通信事業法 外部送信規律** | A/Bテスト Cookie（30日）・Upstash へのIP送信・Resend へのメール送信を、PPまたは外部送信に関するページで開示する。 | `implementation-guide.md §9.5` |

### 重大（リリース前に方針確定が必要）

| # | 項目 | 確認事項 |
|---|---|---|
| 4 | **統計数値の根拠資料** | 本番コンテンツ確定時に「登録企業数・転職成功者数・公開求人件数」の裏付けデータを用意する。実態と乖離すると景品表示法（優良誤認）に該当するリスクあり。 |
| 5 | **利用規約の要否** | フォーム送信による役務提供がある場合、利用規約（免責事項・禁止事項）が必要か法務確認。必要な場合は `/terms` ページを追加しフォームに同意リンクを追加する。 |

### 中程度（運用開始後に対応可）

| # | 項目 | 確認事項 |
|---|---|---|
| 6 | **Testimonials のステルスマーケティング開示** | 謝礼・インセンティブを提供して収集した口コミには「PR」等の表示が必要（2023年10月施行 景品表示法告示）。収集フローと表示方針を別途定めること。 |
| 7 | **特定電子メール法 自動返信メール** | 自動返信メールに送信者の氏名・名称・連絡先住所が記載されているか確認（`implementation-guide.md §8.2`）。 |
