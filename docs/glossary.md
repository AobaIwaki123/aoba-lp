## フロントエンド

### WebGL
ブラウザ上で GPU を使って 3D グラフィックスを描画する技術。通常のウェブページは CPU で HTML/CSS を描画しますが、WebGL は GPU に直接命令を送るため、複雑な 3D 演算をリアルタイムで処理できます。このプロジェクトでは React Three Fiber 経由で使用しています。

### React Three Fiber (R3F)
Three.js（WebGL の 3D ライブラリ）を React コンポーネントとして扱えるようにしたラッパーライブラリ。`<mesh>`, `<sphereGeometry>` のように JSX で 3D オブジェクトを記述できます。

### RSC（React Server Components）
サーバー側でのみ実行される React コンポーネント。DB アクセス・秘密キーの利用が可能で、ブラウザに送る JS バンドルを削減できます。このプロジェクトではページコンポーネントの大部分が RSC です。

### SSG（Static Site Generation）
ビルド時に HTML を生成する手法。リクエストのたびに生成しないため最速です。更新頻度が低いトップページ・About ページに採用しています。

### `'use client'`
Three.js や Framer Motion など、ブラウザの API に依存するコンポーネントに付けるディレクティブ。これがないと SSR（サーバーサイドレンダリング）時にエラーになります。

## バックエンド・インフラ

### Serverless
常時起動のサーバーを持たず、リクエストが来たときだけ関数を実行する仕組み。Vercel・Neon・Upstash はすべて Serverless。管理不要・アイドル時のコストゼロが特徴です。

### Edge Runtime
通常のサーバーより利用者に地理的に近い場所（エッジノード）で実行される軽量な JavaScript ランタイム。Node.js の全機能は使えないかわりにレイテンシが低い。このプロジェクトでは `src/proxy.ts`（A/B テストの Cookie 付与）に使用。

### Server Actions
Next.js の機能。クライアントから直接呼び出せるサーバー側の関数。フォーム送信処理など「ボタンを押したらサーバーで何かする」を API Route なしで実装できます。CSRF 対策が自動で組み込まれています。

### App Router
Next.js 13 以降のルーティング方式。`app/` ディレクトリ以下のファイル構造がそのままURLになります（`app/about/page.tsx` → `/about`）。RSC がデフォルトで使えます。

## データベース

### ORM（Object-Relational Mapper）
SQL を直接書く代わりに、JavaScript/TypeScript のコードでDBを操作できるライブラリの総称。このプロジェクトでは Drizzle ORM を使用しています。

### Drizzle ORM
TypeScript の型情報を活かした軽量な ORM。スキーマ定義からマイグレーションファイルを自動生成でき、全クエリをパラメータ化クエリとして実行するため SQL インジェクションを防ぎます。

### パラメータ化クエリ（Prepared Statement）
ユーザー入力を SQL 文字列に直接埋め込まず、プレースホルダーとして渡す仕組み。SQL インジェクション攻撃を防ぎます。Drizzle ORM を使うと自動的にこの方式になります。

### Neon
PostgreSQL をサーバーレスで提供するクラウドサービス。接続のたびにコネクションプールを意識しなくてよい Serverless Driver を提供しています。Free Tier で 0.5 GB ストレージ利用可能。

### コネクションプーリング
DB への接続を使い回す仕組み。Serverless 環境では関数が都度起動するため接続数が爆発しやすいですが、Neon の Serverless Driver はこれを内部で自動処理します。

### 冪等性キー（Idempotency Key）
「同じ操作を何回実行しても結果が1回と同じ」を保証するためのランダムな文字列。フォーム表示時に生成してフォームデータに含め、DB 側で重複送信を `ON CONFLICT DO NOTHING` で排除します。ネットワーク瞬断や誤操作による二重投稿を防ぎます。

## セキュリティ

### CSP（Content Security Policy）
「このページはどのドメインのリソースだけを読み込んでよいか」をブラウザに伝える HTTP ヘッダー。外部スクリプトの読み込みを制限し、XSS 攻撃の被害を限定します。`next.config.ts` で設定しています。

### HSTS（HTTP Strict Transport Security）
「このドメインには常に HTTPS で接続せよ」とブラウザに記憶させる HTTP ヘッダー。一度訪問したブラウザは次回から自動的に HTTPS を使うようになります。

### CSRF（Cross-Site Request Forgery）
悪意あるサイトから被害者のブラウザを使って正規サービスへのリクエストを送る攻撃。Next.js の Server Actions は `Origin` ヘッダーを自動検証するため、外部サイトからの呼び出しをブロックします。

### XSS（Cross-Site Scripting）
悪意あるスクリプトをページに埋め込む攻撃。React は変数の出力を自動的にエスケープするため大半のケースで防がれます。CSP ヘッダーが追加の防御層になります。

### ハニーポット
フォームに仕込む「目に見えない入力欄」。人間は見えないので入力しませんが、ページを機械的に解析して全フィールドを埋めるボットは入力してしまいます。値があれば成功を偽装してサイレントに棄却します。

### スライディングウィンドウ式レート制限
「直近 N 分間」を常に動く窓で計算するレート制限。固定時間帯リセット方式（例: 毎時0分にリセット）と違い、窓の境界を狙った攻撃が難しくなります。

### x-real-ip
Vercel がリクエストに付与するクライアントの IP アドレスを含むヘッダー。`x-forwarded-for` はリクエストヘッダーに攻撃者が任意の値を付加できますが、`x-real-ip` は偽装できないため、レート制限の IP 識別に使用しています。

### httpOnly Cookie
JavaScript（`document.cookie`）からアクセスできない Cookie の属性。XSS 攻撃でスクリプトが実行されても Cookie を盗まれません。A/B テストの `variant` Cookie に設定しています。

## A/B テスト

### A/B テスト
同じページの複数バージョンをユーザーにランダムに表示し、どちらがコンバージョン率（目標達成率）が高いかを比較する手法。このプロジェクトでは 3 つのデザインコンセプト（A/B/C）を用意しています。

### バリアント
A/B テストにおける「バリエーション」。このプロジェクトでは A（CONNECTED）・B（BOLD）・C（HUMAN）の 3 バリアントがあります。現在は MVP として A と C のみ有効にしています。

### Cookie（クッキー）
ブラウザに保存される小さなデータ。このプロジェクトでは初回訪問時に割り当てたバリアント（A/B/C）を Cookie に保存し、再訪問時も同じデザインを表示するために使います。

## 外部サービス

### Vercel
Next.js を開発した会社が提供するホスティングサービス。GitHub にプッシュするだけで自動デプロイされます。Edge Network により世界中から低レイテンシでアクセスできます。

### Upstash Redis
Redis（高速な Key-Value ストア）を Serverless で提供するサービス。このプロジェクトではレート制限のカウンター管理に使用しています。Free Tier で 10,000 リクエスト/日。

### Resend
メール送信 API サービス。React コンポーネントでメールテンプレートを書けます。SPF/DKIM の設定が簡単で、メールが迷惑メールに入りにくいのが特徴。Free Tier で 3,000 通/月。

### Slack Webhook
指定の Slack チャンネルにメッセージを POST で送れる URL。サーバー側からフォーム受信・エラーを Slack に通知するために使用しています。設定は Slack アプリの管理画面から行います。

### Drizzle Kit
Drizzle ORM に付属するマイグレーションツール。`pnpm db:generate` でスキーマ定義からマイグレーション SQL を生成し、`pnpm db:migrate` で DB に適用します。

## 開発ツール

### Zod
TypeScript で型安全なバリデーションを書くためのライブラリ。スキーマを定義すると TypeScript の型も自動生成されます。フォームのクライアント・サーバー両方で同じスキーマを共有しています。

### React Hook Form
React のフォームライブラリ。再レンダリングを最小限に抑えながらフォームの状態管理とバリデーションを行います。Zod と組み合わせて使用しています。

### Framer Motion
React 向けのアニメーションライブラリ。スクロール連動・ホバーエフェクト・ページ遷移アニメーションに使用しています。

### shadcn/ui
コピー&ペースト型の UI コンポーネント集。npm パッケージとしてインストールするのではなく、コンポーネントのソースコードをプロジェクトに直接追加する方式です（`src/components/ui/` に格納）。カスタマイズが容易。

### Playwright
ブラウザ自動化テストフレームワーク。E2E（エンドツーエンド）テストに使用しています。このプロジェクトでは OG 画像の自動生成にも使用しています。
